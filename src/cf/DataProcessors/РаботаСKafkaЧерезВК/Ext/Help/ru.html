                                 стрИмяТопика                  - Имя топика

                              

                               Возвращаемые значение:

                                               Истина в случае успеха

                                               Ложь  в случае ошибки

//----------------------------------------------------------------------------------------------------------------------------------------

                Subscribe();

                               Возвращаемые значение:

                                               Истина в случае успеха

                                               Ложь  в случае ошибки

                                               Описнаие ошибки можно получить через свойство "ErrorDescription"

                                               Разделы будут назначены автоматически брокером, в зависимости от количества подписанных на

                                               топик получателей с текуей группой(groupID)

                                               В дальнейшем из подписок можно получать сообщения функциями "Consume", "ConsumePool"

//----------------------------------------------------------------------------------------------------------------------------------------                  

                Unsubscribe();

                               Возвращаемые значение:

                AddRecordToTopicPartitionList(стрТопик, чРаздел, чСмещение);

                                 чСмещение                        - Смещение, с которого будет начато чтение, Неопределено - для последнего смещения закоммиченного в брокер

                              

                               Возвращаемые значение:

                                               Истина в случае успеха

                                               Ложь  в случае ошибки

                                               Описнаие ошибки можно получить через свойство "ErrorDescription"

                                              

                               Описание:

                                               Добавляет топик-раздел-(смещение) в список на назначение (для использования функцией "Assign")

                                               (заполняет внутренний список внутри компоненты)

//----------------------------------------------------------------------------------------------------------------------------------------  

                Assign();

                               Возвращаемые значение:

                                               Истина в случае успеха

                                               Ложь  в случае ошибки

                                               Описнаие ошибки можно получить через свойство "ErrorDescription"

                                              

                               Описание:

                                               Выполняет назначение топиков-рвзделов-(смещений) из списка, полученного вызовами функции "AddRecordToTopicPartitionList"

                                               При использовании этого метода не происходит ребалансировки партиций между получателями одной группы, получатель читает

                                               только из назначенных функцией партиций

                                               В дальнейшем из подписок можно получать сообщения функциями "Consume", "ConsumePool"

//----------------------------------------------------------------------------------------------------------------------------------------  

                Unassign();

                               Возвращаемые значение:

                                               Истина в случае успеха

                                               Ложь  в случае ошибки

                                               Описнаие ошибки можно получить через свойство "ErrorDescription"

                                              

                               Описание:

                                               Сбрасывает назначения разделов, сделанные функцией "Assign"

                                               Получение останавливается (методы "Consume", "ConsumePool" - всегда завершаются таймаутами) после вызова функции

//----------------------------------------------------------------------------------------------------------------------------------------                  

                Consume(чТаймаут);

                                 чТаймаут - Время в миллисекундах после которого происходит ошибка ERR__TIMED_OUT, если данные не были получены

                              

                               Возвращаемые значение:

                                               Истина в случае успеха

                                               Ложь  в случае ошибки

                                               Описнаие ошибки можно получить через свойство "ErrorDescription"

                                              

                               Описание:

//----------------------------------------------------------------------------------------------------------------------------------------                                                                 

                ConsumePool(чТаймаут, чКоличествоСообщений, чКоличествоОшибокДляПрерывания);

                                 чТаймаут                                                            - Время в миллисекундах после которого происходит ошибка ERR__TIMED_OUT, если данные не были получены (для каждого сообщения отдельно)

                                 чКоличествоСообщений                                 - Количество сообщений, которое необходимо получить

                                 чКоличествоОшибокДляПрерывания         - Количеств ошибок(например таймаутов), после которого выполнение будет прервано

 

                              

                               Возвращаемые значение:

                                               Истина в случае успеха

                                               Ложь  в случае ошибки

                                               Описнаие ошибки можно получить через свойство "ErrorDescription"

                                              

                               Описание:

                                               Пытается получить список сообщений и в случае успеха, сохраняет их во внутренний список компоненты

                                               В процессе получения возможно таймауты т.к. если мы читаем из нескольких разделов - в каком-либо

                                               могут закончится данные, а вдругих нет - соответсвенно при чтении из раздела, где данные закончились

                                               мы получим ошибку ERR__TIMED_OUT - поэтому при чтении из нескольких разделов имеет смысл не прерываться после превого таймаута

                                               таймауты также могут быть из-за перебалансировок паритиций между получателями

                NextRecord();

                               Возвращаемые значение:

                                               Истина в случае наличия записи

                                               Ложь  в случае отсутствия записи

                                              

                               Описание:

                                               Выполняет перебор записей из внутреннего списка компоненты (похоже на Выборка.Следующий())

                                               Запись может быть прочитана из свойств объекта:

                                                              

                                                               RecordTopic                                                        - Топика из которого ьыла прочитана запись

                                                               RecordPartition                                  - Раздел из которого была прочитана запись

                                                               RecordOffset                                                       - Смещение записи         

                                                               RecordTimestamp                                              - Временная метка записи                            

                                                               RecordKey                                                           - Ключ записи    

                                                               RecordValue                                                       - Значение записи (двоичные данные)

                                                               RecordValueString                                             - Значение записи (строка)            

//----------------------------------------------------------------------------------------------------------------------------------------  

                ResetRecordCounter();

                                               Заголовок может быть прочитан из свойств объекта:

                                                              

                                               HeaderKey                           - Имя заголовка

                                               HeaderValue        - Значение заголовка

                                               Если сообщения содержат бинарные данные - JSON сформировать не удастся
